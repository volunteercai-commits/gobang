# 分支策略说明

## 🌳 分支结构

### main 分支
- **用途**: 生产环境分支，只包含稳定的发布版本
- **保护**: 只能通过Pull Request合并
- **内容**: 当前包含v1.0.0版本

### develop 分支
- **用途**: 开发主分支，集成所有新功能
- **保护**: 开发人员的主要工作分支
- **内容**: 包含最新的开发代码

### release/* 分支
- **用途**: 发布准备分支，用于版本发布前的最后调整
- **命名**: release/v1.0, release/v1.1 等
- **内容**: 从develop分支创建，用于bug修复和版本准备

## 🏷️ 版本标签

### v1.0.0 (当前版本)
- **功能**: 五子棋游戏正式版
- **特性**:
  - 人机对战和人人对战
  - Minimax+Alpha-Beta剪枝AI算法
  - 专业开局库和高级威胁检测
  - 悔棋功能和计分系统
  - 响应式设计，支持移动端
  - 现代化UI和Canvas绘制

## 🔄 工作流程

### 日常开发
1. 从 `develop` 分支创建功能分支
2. 在功能分支上开发
3. 完成后合并回 `develop` 分支

### 版本发布
1. 从 `develop` 创建 `release/vX.X` 分支
2. 在release分支进行最后的bug修复
3. 测试完成后合并到 `main` 分支
4. 创建版本标签 `vX.X.X`

### 热修复
1. 从 `main` 分支创建 `hotfix/*` 分支
2. 修复后合并到 `main` 和 `develop` 分支

## 📋 分支命名规范

- **功能分支**: `feature/功能名称`
- **修复分支**: `bugfix/问题描述`
- **热修复**: `hotfix/问题描述`
- **发布分支**: `release/v版本号`

## 🚀 部署说明

- **生产环境**: 从 `main` 分支部署
- **测试环境**: 从 `develop` 分支部署
- **发布脚本**: `./deploy.sh`
